# EIP 會議室預約系統重構說明

本文件記錄了本次針對 `admin/meeting_room.php` 的完整重構：從傳統表單式預約，改為**視覺化週次時間格**介面，並新增**取消預約**功能。

---

## 📋 一、重構總覽

| 項目 | 修改前 | 修改後 |
|------|--------|--------|
| 預約方式 | 手動填寫日期時間表單 | 點選週曆時間格，自動填入日期與時段 |
| 時間呈現 | 下拉/文字輸入框 | 7 天 × 11 時段的可視化週曆表格 |
| 衝突視覺 | 送出後才顯示錯誤 | 已預約格直接顯示顏色與預約人 |
| 取消預約 | ❌ 無此功能 | ✅ 本人預約格顯示取消按鈕 |
| 多時段 | 分次送出 | 點選多格連續時段，一次送出 |

## 🗓️ 二、視覺化週曆介面

### 介面結構

```
┌─────────────────────────────────────────────────────────┐
│ 1. 預約會議室：[A1 視訊會議室 ▼]  ℹ 點選時段出現打勾符號   │
│ << 上個月  < 上週  [ 回到今天 ]  下週 >  下個月 >>        │
├──────────┬────────┬────────┬────────┬───...───┤
│  日期    │星期一  │星期二  │星期三  │ ... │星期日│
├──────────┼────────┼────────┼────────┼───...───┤
│08:00-09:00│ 已過期│  空    │  空    │         │
│09:00-10:00│ 已過期│ 7710   │  空    │         │
│           │       │ 何佳融 │        │         │
│  ...      │       │        │        │         │
└──────────┴────────┴────────┴────────┴─────────┘
                      [ ✔ 確認預約 ]
```

### 格子狀態與顏色

| 顏色 | 狀態 | 說明 |
|------|------|------|
| 🩶 灰色 `#d0d0d0` | 已過期 | 過去的日期/時段，不可點選 |
| 💛 淡黃色 `#fff3b0` | 他人預約 | 顯示分機號 + 姓名，不可操作 |
| 🟡 橘黃色 `#ffd87a` | **本人預約** | 顯示分機號 + 姓名 + 🗑️ 取消按鈕 |
| 💚 淡綠色 `#c8f0d4` | 空位 | 可點選預約，hover 時變深綠 |
| 💙 藍色 `#1d6cb5` | 已選取 | 點擊後出現 ✔，再次點擊取消選取 |

### 週曆導覽

| 按鈕 | 動作 |
|------|------|
| `<< 上個月` | `week_offset - 4` |
| `< 上週` | `week_offset - 1` |
| `回到今天` | `week_offset = 0` |
| `下週 >` | `week_offset + 1` |
| `下個月 >>` | `week_offset + 4` |

## 📅 三、預約流程

```
選擇會議室
    ↓
瀏覽週曆，點選空位格（💚）
    ↓ 格子變藍（💙），出現 ✔
可繼續點選同日連續時段
    ↓ 下方顯示「已選取 N 個時段」
按下「確認預約」
    ↓
彈出確認 Modal（顯示會議室/日期/時段摘要）
    ↓
填寫「會議目的/用途」
    ↓
按「送出預約」→ POST 表單送出
    ↓
成功：頁面顯示綠色成功訊息
失敗：頁面顯示紅色錯誤訊息
```

> **限制**：一次只能預約**同一天的連續時段**。若選取跨日或非連續時段，送出時會彈出警告。

## 🗑️ 四、取消預約流程

```
找到橘黃色格子（本人預約 🟡）
    ↓
點擊格中的 🗑️ 取消 按鈕
    ↓
彈出「取消預約確認 Modal」
顯示：會議室 / 日期（含星期） / 時段 / 用途
    ↓
按「確認取消」→ POST 表單送出（action=cancel_room）
    ↓
成功：頁面顯示「🗑️ 預約已成功取消！」
失敗：顯示紅色錯誤訊息
```

> **安全限制**：只有**本人預約**的時段才顯示取消按鈕；他人的預約、已過期的時段皆無法取消操作。

## ⚙️ 五、PHP 後端核心邏輯

### 週曆資料結構
```php
// GET 參數
$week_offset   = intval($_GET['week_offset'] ?? 0); // 週次偏移
$selected_room = $_GET['room'] ?? $rooms[0];        // 選取的會議室

// 計算本週星期一為起點
$week_start->modify('Monday this week');
$week_start->modify("{$week_offset} weeks");

// 預約映射表 [日期][時段起始] => 預約資訊
$reservations_map[$date][$slot_start] = [
    'id'       => $r['id'],
    'realname' => $r['realname'],
    'ext_no'   => $r['ext_no'],
    'purpose'  => $r['purpose'],
    'username' => $r['username'],
    'is_mine'  => ($r['username'] === $user['username']),
    'start'    => $s_hour,
    'end'      => $e_hour,
];
```

### 新增預約（`action=reserve_room`）
- 補齊時間秒數（`14:00` → `14:00:00`）
- 驗證日期不可為過去日期
- 衝突檢查：`NOT (end_time <= start OR start_time >= end)`
- MySQL：`INSERT INTO room_reservations`
- Session：`$_SESSION['mock_reservations'][] = [...]`

### 取消預約（`action=cancel_room`）
- MySQL：`DELETE FROM room_reservations WHERE id = ? AND username = ?`（只能刪除自己的）
- Session：`unset` 對應陣列元素後 `array_values` 重新索引

## 🧪 七、驗證測試情境

1. **週曆切換**：點擊上週/下週/回到今天，確認週曆日期正確滾動。
2. **會議室切換**：下拉選單切換 A1/A2/A3，確認對應的預約資料獨立顯示。
3. **預約測試**：
   - 點選單一空位格 → 確認 Modal 顯示正確 → 填寫用途送出 → 確認成功訊息與格子更新。
   - 點選同日多格連續時段 → 確認 Modal 顯示合併後的時段（如 09:00 ～ 11:00）。
   - 嘗試選取跨日格子 → 應顯示「一次只能預約同一天的連續時段」警告。
4. **取消測試**：
   - 找到本人預約的橘黃色格，點擊「取消」按鈕 → Modal 顯示正確資訊 → 確認取消 → 格子恢復空位綠色。
   - 確認他人預約的淡黃色格無取消按鈕。
5. **衝突防護**：嘗試預約已佔用的時段 → 應顯示衝突錯誤訊息。
6. **過去日期防護**：嘗試預約今天之前的日期（表單直接送出）→ 應顯示日期錯誤。
